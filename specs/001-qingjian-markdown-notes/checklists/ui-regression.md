# UI 回归手工检查清单

**Purpose**: UI 行为回归验证（暗色模式/键盘/VoiceOver/错误语义一致/跨端体验一致）  
**Created**: 2025-12-25  
**Feature**: [spec.md](../spec.md)

---

## 跨端一致性

- [ ] macOS 与 iOS 使用相同术语（Repo/笔记/目录/冲突等）
- [ ] macOS 与 iOS 的错误提示语义一致（例如"Repo 不可用"在两端展示相同含义）
- [ ] macOS 与 iOS 的 Repo 列表展示顺序一致（按元数据中的自定义排序）

## 暗色模式

- [ ] macOS：View 渲染在暗色模式下可读性良好（对比度、代码块背景）
- [ ] macOS：编辑器在暗色模式下可读性良好（光标/选中/行号）
- [ ] macOS：预览在暗色模式下可读性良好
- [ ] iOS：View 渲染在暗色模式下可读性良好
- [ ] iOS：目录树/列表在暗色模式下可读性良好
- [ ] 暗色/亮色切换时，缓存失效与重渲染正常

## 键盘优先工作流（macOS）

- [ ] 可用键盘完成"打开 Repo → 选择笔记 → View/编辑 → 保存"主流程
- [ ] 焦点在目录树/笔记列表/编辑器/预览之间可用 Tab/快捷键切换
- [ ] Vim 模式下常见操作（移动/选择/删除/撤销/重做/查找）响应正确
- [ ] 输入法与 Vim 模式共存时无异常（切换输入法不干扰 Vim 状态）

## VoiceOver / 辅助功能

- [ ] macOS：VoiceOver 可读取 Repo 列表、目录树节点、笔记标题
- [ ] macOS：VoiceOver 可访问编辑器内容
- [ ] iOS：VoiceOver 可读取 Repo 列表、目录树节点、笔记标题
- [ ] iOS：VoiceOver 可访问 View 渲染后的内容

## 错误与降级

- [ ] Repo 路径不可用时，macOS 与 iOS 都显示清晰提示与恢复入口
- [ ] 笔记文件损坏/编码不可识别时，显示降级提示而非崩溃
- [ ] 图片资源缺失或在线图片不可访问时，显示清晰的占位/错误提示
- [ ] iCloud 同步冲突时，macOS 与 iOS 都能看到冲突状态并引导处理

## 性能体验（主观）

- [ ] 打开 Repo 后目录树加载无明显卡顿
- [ ] 切换笔记进入 View 模式响应迅速
- [ ] macOS 编辑器输入无明显延迟
- [ ] macOS 预览更新"感觉即时"
- [ ] 滚动流畅（View/编辑器/预览）
